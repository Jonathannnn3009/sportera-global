How It Works
1

Player / Team Registers

A player or team fills the registration form on the main website. Admin receives the details and creates their portal account manually via the Admin Panel.

2

Admin Creates Account

Admin goes to the portal → Create Accounts section, enters name + email. Firebase sends a password reset email to the player/team so they set their own password.

3

Player / Team Logs In

They open the portal from the website footer, enter their email and password, and land on their personal dashboard with all their data.

4

Admin Manages Everything

Admin enters match stats, assigns awards, approves squad changes, and controls whether registration and squad editing windows are open or closed.

What Each Portal Can See & Do

Player Dashboard

  • Profile: name, city, position, team, DOB, Player ID
  • Stats: matches played, goals, assists, yellow & red cards
  • Tournament history with status (Champion, Runner-Up etc.)
  • Match-by-match history with goals and cards per game
  • Awards won: Golden Boot, Best Player, Golden Glove
  • Read-only — player cannot edit anything

Team Dashboard

  • Profile: team name, city, coach, contact email
  • Squad grid showing all players with Approved / Pending / Rejected status
  • Add player form (only visible when admin opens the editing window)
  • Tournament history with division and final position
  • Match results with scores
  • Locked notice shown when admin closes squad editing

Admin Panel

  • View all registered players and teams
  • Input match stats per player (goals, assists, cards)
  • Approve or reject squad addition requests from teams
  • Assign awards (Golden Boot, Best Player, etc.)
  • Toggle registration window open / closed
  • Toggle squad editing window open / closed
  • Create player and team login accounts

Important Notes
Admin URL is not public. The admin panel link is not shown anywhere on the website. Only you know the URL: portal/admin-portal.html. Bookmark it.
Security. Every user can only see their own data. Firebase security rules block cross-account access. Only the admin UID can write or read all records.
Real accounts. When you go live, delete these test accounts from Firebase Authentication and create real accounts for actual players and teams via the Admin Panel.
Squad editing window. Teams can only add players when admin has toggled "Squad Editing" ON. Toggle it OFF to lock the window before a tournament starts.
